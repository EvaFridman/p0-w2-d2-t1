# p0-w2-d2-t1 · Сетка из колонок и строк

> Тип: короткая отработка · Репозиторий: один отдельный

## Задача

Вчера флекс раскладывал элементы в одну линию. Grid раскладывает сразу в двух измерениях —
строки и колонки одновременно. Сегодня набиваешь руку на явной сетке: задаёшь треки колонок и
строк, расставляешь по ним блоки, ставишь промежутки через `gap`.

```
grid-template-columns: 1fr 2fr 1fr
┌──────┬─────────────┬──────┐
│  A   │      B      │  C   │   колонки = треки
├──────┼─────────────┼──────┤   gap между ячейками
│  D   │      E      │  F   │
└──────┴─────────────┴──────┘
fr — доля свободного места, не пиксель
```

## Что нужно сделать

`index.html` с одним grid-контейнером и блоками внутри (пронумерованные квадраты подойдут),
подключённый `style.css`. Дальше — только CSS: получить раскладки ниже. Каждую следующую
делаешь, предыдущую закомментировав в CSS с пометкой номера релиза (`/* release 2 */`), чтобы в
финальном PR были видны все шаги.

Что нужно получить:

- три колонки одинаковой ширины через `fr`, между ними `gap`;
- три колонки в пропорции 1 / 2 / 1 (средняя вдвое шире);
- колонки в пикселях и `fr` вперемешку (например боковая 200px, центр `1fr`);
- задать ещё и строки (`grid-template-rows`), получить сетку 3×2;
- сократить запись трёх одинаковых колонок через `repeat()`.

## Критерии приёмки

* Контейнер — `display: grid`, расстояние задано через `gap`, не отступами у блоков.
* Получены все раскладки, повторяющиеся колонки записаны через `repeat()`.
* Можешь объяснить на словах, чем `fr` отличается от `%` и от `px` при задании трека.
